BATOCO

Overview

Batoco was initially built after a request to convert zmakebas into PHP to allow it to work on a webserver that didn’t allow the use of exec.

Since then it has grown to support more machines in the ZX family. It is functionally equivalent to zmakebas.c though it is mostly written from scratch. It has all the extra features, like labels and shortcuts to embed UDGs.

How to run it

Batoco needs PHP installed to run. Most webservers and unix-based machines will already have this installed. Windows-based machines might need to install it.

Windows

The PHP file can be called from the commandline in Windows via

batoco.PHP

For example:

c:\PHP\PHP batoco.PHP .\Test_Files\GOTO_Test_Labels.bas -v -L -n GTO-TEST -o GOTOTestPHP.tap

Unix/Linux

In unix you should be able to just run batoco.PHP

For example:

batoco.PHP .\Test_Files\GOTO_Test_Labels.bas -v -L -n GTO-TEST -o GOTOTestPHP.tap

Webserver

It can also be called via a URL using POST or GET to send the arguments

e.g. http://localhost/batoco.php?input=.\Test_Files\ZMAKEBAS_Test_Labels.bas&l=on&n=ZMB-TEST&o=ZMAKEBASTestPHP.tap

How to use it

By default Batoco will target a SPECTRUM machine and produce an output file called output.tap. Further options can be given on the commandline to give more control.

Machines supported

Machine

Description

ZX80

The most basic of the range.

ZX81

Advance on the ZX80, lots more keywords. Characters have different places in the character set.

LAMBDA1

ZX81 clone with many extra keywords and different graphics.

LAMBDA2

Same as Lambda 1 but with extra keywords INK,PAPER and BORDER

NF300

Lambda clone, contains the extra keywords READ,RESTORE and DATA, same as a CAC-3

SPECTRUM

16K to 128K basic also covers identical clones

TIMEX

Same as Spectrum except characters {,},|,~,© are missing and new keywords ON ERR,STICK,SOUND,FREE and RESET are available.

PLUS3

Same as Spectrum, generates a PLUS3 file header.

NEXT

Adds many new keywords, generates a PLUS3 file header.

Differences to zmakebas

Hex numbers -

Like zmakebas you can use Hex numbers after a BIN keyword. These are expected to start 0x. However, these are translated back into a binary representation in the outputted file.

For example:

  BIN 0xAA4E will appear in the output as BIN 1010101001001110

Commandline parameters -

The input filename is given as the first parameter, not the last parameter

For example:

zmakebas -l inputfile.bas

PHP batoco.php inputfile.bas -l

Spanish characters

Spanish characters Ñ,¿,¡,ñ are supported for Spanish ROM machines

TZX

SPECTRUM and TIMEX targets can be requested to output TZX files rather than the default TAP files.

Split lines

You can also split long lines by placing a single backslash ("\") character at the end of the line you wish to wrap:-

For example:

   1120 IF lives>3 AND score>20000 THEN \

        GOSUB 4200

Would output as

1120 IF lives>3 AND score>20000 THEN GOSUB 4200

Label mode

Using the "-l" command line option allows you to write labelled code. Batoco will produce the appropriate line-numbered output. Program labels must always start with an @ character, and must terminated with a colon (':').

The “-s” command line option specifies the number to start numbering the code with.

The “-i” command line option sets the increment between line numbers.

For example:

Using the commandline options -l -s 231 -i 5 and the code file below

       CLS

       PRINT "HELLO WORLD."

    @loop1:

       PRINT "THIS IS A LOOP"

       GOTO @loop1        

Would output

231 CLS

236 PRINT “HELLO WORLD”

241 PRINT “THIS IS A LOOP”

246 GOTO 241

# Comment lines and blank lines

The # sign has two purposes, in the first case it acts as a comment line. Any contents in a comment line are ignored in the final output. Blank lines are also ignored.

For example:

# This is a comment I will not appear in the output

10 PRINT “HELLO WORLD”

20 GOTO 10

The above will output as

10 PRINT “HELLO WORLD”
20 GOTO 10

#! Preprocessor lines

Batoco operates a basic preprocessor. A preprocessor line begins with  #! These lines will affect compilation. This is based on ZX81 BASIC to P-File Converter by maziac. The commands recognised are as follows:

#!basic-start: [N] - This will cause the program to automatically start at the specified start line.

For example:

#!basic-start:100

which will start the BASIC program at line 100 after loading.

#!dfile: [TEXT] - Writes a line to the D_FILE (ZX81, Lambda 8300)

Each time the line #!dfile: [TEXT] is used it will add a line to the D_FILE with the contents in [ ]. If you don’t add any text an empty line will be added.

For example:

#!dfile:   HELLO

/#!dfile:

/#!dfile:   WORLD

Would add

   HELLO

   WORLD

#!dfile-collapsed: - To save space in a program for the ZX81 this command collapses the D_FILE

               

#!machine: [TARGET LIST] - This allows you to target specific machines, everything is included up to #!end-machine: Valid targets are SPECTRUM, ZX80, ZX81, PLUS3, TIMEX, NEXT, LAMBDA1, LAMBDA2

For example:

10 PRINT “I AM A “;

#!machine:ZX81

20 PRINT “ZX81”

 #!end-machine:

#!machine:SPECTRUM

20 PRINT “SPECTRUM”

#!end-machine:

Will produce a program that prints I AM A ZX81 on a ZX81 and I AM A SPECTRUM on a Spectrum.

Multiple machines can be listed #!machine:ZX81,LAMBDA1,ZX80

#!machine-not: [TARGET LIST] - This allows you to exclude specific machines, everything is included up to #!end-machine: Valid targets are SPECTRUM, ZX80, ZX81, PLUS3, TIMEX, NEXT, LAMBDA1, LAMBDA2

For example:

10 PRINT “I AM NOT A “;

#!machine-not:ZX81

20 PRINT “ZX81”

 #!end-machine:

#!machine-not:SPECTRUM

20 PRINT “SPECTRUM”

#!end-machine:

Will produce a program that prints I AM NOT A ZX81 on a ZX81 and I AM NOT A SPECTRUM on a Spectrum.

NOTICE: #!machine: and #!machine-not: commands can not be nested.

#!warning: [MESSAGE] - Prints a warning message on the commandline.

For example:

#!machine:ZX81

#!warning: This game may run very slowly on this target.

#!end-machine:

If you are building for a ZX81 it would output:

Warning: This game may run very slowly on this target. On the commandline.

#!error: [MESSAGE] - Prints an error message on the commandline and quits

For example:

#!machine:ZX81

#!error: This code will not run on a ZX81

#!end-machine:

If you are building for a ZX81 it would output:

Warning: This code will not run on a ZX81. On the commandline and Batoco would quit.

Inverse Video

When building for ZX80,ZX81,LAMBDA1 and LAMBDA2 targets inverse video (white on black for ZX80 and ZX81, or black on white for LAMBDA1 and LAMBDA2) text can be included in your program by using the percent symbol ('%') as a character prefix.

For example:

  10 PRINT "%I% %a%m% %i%n%v%e%r%t%e%d I am not"

Escape Codes, Block Graphics and UDGs

In BASIC, the quote symbol (") is used to delimit strings of text. If you want a quote symbol to appear in the middle of a string, you must therefore prefix it with a backslash ("\") escape character.

For example:

  10 PRINT "He said, \"Hello.\""

Block graphics

The full range of block graphics characters can easily be inserted into a program by using the backslash ("\") escape character followed by a 2 character "ASCII-art" style rendition of the desired graphics character.

For example:

  10 PRINT "\:' \':"

UDGS

On the Spectrum/Timex targets a letter following a \ represents that UDG.

For example:

\a\b\c

Would print the first 3 UDGs.

Option

Meaning

-h

Show usage help.

-o output-file

Write the output to the named file (default="out.p").

-l

Use labels mode (see the section on labels, below).

-s line

In labels mode, sets the starting line number (default=10).

-i

In labels mode, set the line number increment (default=2).

For example, zxtext2p -l -s 100 -i 5 -o mygame.p gamesrc.txt will read the file gamesrc.txt, which must contain labelled code rather than code with line numbers, and will produce a .P file called mygame.p The first line will be numbered 100, the next 105, and so on.
